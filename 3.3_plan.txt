# 暴叔 AI 3.3 开发计划：企业微信接入与架构升级

## 1. 核心目标
- 接入企业微信（微信客服路径），实现真实微信用户交互。
- 引入 Redis 消息缓冲区，解决用户“连珠炮”式输入导致的状态冲突。
- 优化 Web 端体验，允许用户在 AI 响应期间持续输入。

## 2. 技术路线图
### 第一阶段：消息防抖与并发控制 (当前重点)
- **引入 Redis**：本地与服务器安装并配置 Redis 环境。
- **消息合并器 (Buffer)**：
    - 收到请求后先存入 Redis，设置 2-3 秒过期时间。
    - 若期间有新消息，追加内容并刷新过期时间。
    - 过期回调后，合并消息并触发一次 LangGraph 运行。
- **锁机制**：针对 session_id 设置分布式锁，防止同一个用户触发多个并发运行。

### 第二阶段：企业微信对接
- **回调接口实现**：支持 GET 验证与 POST 消息解密（XML 格式）。
- **异步回复**：收到消息后立即返回 200 OK，由后台 Worker 跑完 AI 后通过 API 推送结果。
- **用户识别**：使用 `external_userid` 替代本地生成的 uuid。

### 第三阶段：持久化
- 引入轻量级数据库（如 PostgreSQL/SQLite）记录 CustomerProfile 和长期历史。

## 3. Web 端优化
- 解除前端输入框锁定，采用消息队列展示，提升交互流畅度。
